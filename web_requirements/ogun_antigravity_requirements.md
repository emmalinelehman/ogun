# Ogun USA Website Requirements + Style Guidelines

## Project Goal

Build a polished, modern website for **Ogun USA**, a defense technology startup focused on high-velocity engineering, computational design, AI-assisted optimization, and additive manufacturing. The website should communicate credibility to potential customers, employees, collaborators, and investors.

The visual direction should feel like **defense tech innovation**, taking inspiration from companies such as Saronic, Anduril, Helsing, Neural Concept, nTop, and SimScale. The current website plan calls for a black-background design with white sans serif typography, engineered product visuals, and video-based hero content.

## Site Map

The website should include four primary pages:

1. **Home**
2. **Mission**
3. **Join**
4. **Contact**

The main navigation should be simple and minimal. Use **OGUN USA** as the primary brand mark in the header, with a menu or navigation links depending on screen size. Use a shared header and shared footer across pages.

---

# Global Design Direction

## Overall Visual Style

The site should feel:

- Dark
- Technical
- Sharp
- Industrial
- Premium
- Mission-driven
- Minimal but not empty
- More defense/aerospace startup than consumer product brand

Use a **black or near-black background** as the dominant visual foundation. Typography should be white or light gray. Accent color may be used sparingly, but the core identity should remain restrained and monochrome.

The design should use:

- White sans serif font on black
- Video or imagery of engineered products
- Sansation font for the Ogun logo if available

## Typography

Use **Sansation** for the Ogun USA logo if available.

For body and interface typography, use a clean, narrow or technical sans serif. Acceptable options:

- Sansation
- Inter
- Helvetica Neue
- IBM Plex Sans Condensed
- Space Grotesk
- Saira Condensed

Recommended hierarchy:

- Large hero text: bold, condensed, uppercase or title case
- Section headers: clean, tall, high-contrast
- Body copy: readable, restrained, not overly stylized
- Labels/kickers: small uppercase text with increased letter spacing

Avoid overly playful, rounded, decorative, or consumer-app typography.

## Color Palette

Primary colors:

- Background: `#000000` or `#050505`
- Section background: `#0A0A0A`
- Card/background contrast: `#111111`
- Primary text: `#FFFFFF`
- Secondary text: `#B8B8B8`
- Muted text: `#777777`
- Borders/dividers: `#2A2A2A`

Optional accent colors:

- Electric blue, inspired by the hero plume/video: `#00AEEF`
- Military green or acid green should be used cautiously and only if needed for small CTAs

Do not overuse color. The site should rely on contrast, typography, motion, product imagery, and spacing.

# Page Requirements

### Purpose

The home page should immediately position Ogun USA as a high-tech aerospace and defense company building a new engineering and manufacturing system.

### Hero Section

The hero should use a full-width or large-format video/image background showing an engineered product, simulation, plume, manufacturing, or aerospace/defense-related visual.

Required hero content:

- Brand: **OGUN USA**
- Primary headline: **High Velocity Engineering**
- Supporting line: **Design, optimize, and build at the speed of thought**
- Optional category text: **Aerospace & Defense Products**

The current video may be used as a placeholder, but it should be easy to replace later because the current version contains technical errors.

Create a section titled:

**High Velocity Engineering**

Include the following capability content:

**Capabilities**  
The proprietary High Velocity Engineering system significantly accelerates turning creativity into aerospace and defense products.

The platform integrates CAD design with embedded CFD and FEA modeling and optimization using generative AI. Simulated results are closely tracked to real-world validation for continual improvement.

**Results:**

- Lower cost to experiment
- Remove process friction and focus on insights that improve performance
- Fast learning, iteration, and improvement

Then include three capability columns:

**Computational Design**  
Unifies engineering tasks and workflows.

**AI Optimization**  
Enables interactive what-if scenarios and optimization.

**Scalable Manufacturing**  
Rapid, mass-customizable production with precision.

This section should use a three-column layout on desktop and stack vertically on mobile. Include technical product imagery or placeholders for each capability.

### Product Development Section

Create a section titled:

**Aerospace & Defense Products**

Use this copy:

We are delivering on US defense goals for mass-produced, low-cost, modular solutions.

We are currently iterating a product concept that reinvents optimizing combustion and gas plume management for improved performance.

**Join the Mission**

# Layout Guidelines

## Desktop

Use a wide, cinematic layout. The site should look best on a 16:9 display.

Recommended max content width:

- Standard content: 1200px to 1440px
- Hero: full width
- Capability/product sections: wide grid

Use generous spacing and avoid crowding. The site should feel precise and deliberate.

## Mobile

The site must be fully responsive.

Mobile requirements:

- Header collapses into a menu
- Hero video/image remains readable
- Text overlays must remain legible
- Capability columns stack vertically
- Footer stacks into readable sections
- Job description remains easy to scan
- Contact form fields fill available width

---

# Interaction + Motion Guidelines

Use subtle motion only.

Recommended:

- Slow fade-in on page load
- Slight text reveal on hero
- Hover states on buttons and links
- Gentle image scale on hover
- Video background with dark overlay

Avoid:

- Flashy animations
- Parallax that hurts readability
- Overdesigned loading sequences
- Anything that feels like a consumer SaaS landing page

# Technical Requirements

## Build

Create a clean, maintainable website using a modern front-end stack. Preferred options:

- React
- Next.js
- Vite + React

Use reusable components:

- Header
- Footer
- Hero
- Section container
- Capability grid
- Product media block
- Job posting
- Contact form
- Button

## Accessibility

Minimum requirements:

- Semantic HTML structure
- One clear H1 per page
- Logical heading order
- Sufficient color contrast
- Keyboard-accessible navigation
- Visible focus states
- Alt text for meaningful images
- Captions or accessible fallback for video if required
- Contact form labels tied to inputs
- Do not rely on color alone to communicate meaning

## Performance

- Optimize video assets
- Provide fallback poster images
- Lazy-load non-critical media
- Avoid overly large background videos on mobile
- Use compressed images
- Ensure the site loads quickly despite the visual-heavy design

## SEO Basics

Each page should include:

- Unique page title
- Meta description
- Open Graph title/description
- Basic favicon support
- Clean URLs:
  - `/`
  - `/mission`
  - `/join`
  - `/contact`
